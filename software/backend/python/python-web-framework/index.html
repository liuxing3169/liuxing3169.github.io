2020 排名 Top13

Full-stack frameworks

全栈框架或企业框架是将库配置为无缝协同工作的一体化解决方案。它支持后端服务，前端接口和数据库的开发。全栈框架可提供开发人员构建应用程序所需的一切。Python提供了多个完整的框架。

  • Django
  • Pyramid
  • TurboGears
  • Web2py

Microframeworks

微框架或简约的Web应用程序框架缺少完整框架的大部分功能，例如Web模板引擎，身份验证功能，帐户，授权，输入验证和输入环境卫生。微框架试图仅提供构建应用程序所需的组件集。它还可能专注于为一个特定领域提供必要的功能。

  • Flask
  • Bottle
  • CherryPy
  • Falcon
  • Hug
  • FastAPI

Asynchronous frameworks

异步框架是一种相对较新的Python框架。这是一个微框架，使开发人员能够处理大量并发连接。异步框架使用非阻塞套接字，feed on Python’s asyncio library。

  • Sanic
  • Tornado

借用 @dongweiming 的一些经验，

选择 Web 框架时应遵循的原则

  1. 选择更主流的框架。 因为它们的文档更齐全，技术积累更多，社区更繁盛，能得到更好的支持。
  2. 关注框架的活跃情况。 关注项目的更新频率，Issue 和 Pull Request 的相应情况。如果一个项目已经很长时间没有更新了，或者有一堆的问题需要解决但没有得到回应，就不应该将这样的框架放在生产环境中。
  3. 确认选择的框架是否足够满足需求。没有最好的框架，只有最合适的框架。 你所选择的Web框架不仅需要满足当前的需求，也要充分考虑项目发展一段时间之后的情况，即为前瞻性。如果在选择时有个人喜好这样的因素，需要确认自己有能力对选择的Web框架提供支持，避免盲目选择而导致将来推倒重来的情况。
  4. 注意媒体时效性。在做选择的时候可能会参考网络上的一些文章，但需要注意其发表时间。举个例子，看了一瓶2012年的博客，里面说应该选择 A 而不是 B,并给出了多个理由，然而现在的情况可能已经发生了变化：B 经过很久的努力已经做得更优秀或者在2012年之后出现了更优秀的 C。
  5. 客观看待媒体的观点。媒体的观点不一定总是正确的（或者不是全部正确），如果不是官方的说明，就应该保持怀疑和谨慎的态度，取其精华去其糟粕，切勿完全拿来主义，应该真正做到实践验证之后再做决定。

对于上述Web框架，可以参考以下几点建议：

  1. 如果是未来供演示或者不太考虑长期发展，只是需要用到数据库的 CURD 操作、写 REST API 之类的简单型应用，框架的选择非常宽松，用着顺手即可。
  2. 如果你初学 Web 框架，建议选择 Flask 作为入门框架，学习曲线相对 Django 而言要低很多。等熟悉 Flask 之后再学习 Django ，就会容易很多。
  3. Pyramid 和 Django 都是面向大型应用的， Pyramid 更灵活，开发者空间大得多，值得考虑。
  4. 网站性能出现问题时，往往问题不只是出在 Web 框架或编程语言上，做选择时在环境中按实际产品逻辑测试一段时间即可得出结论，但是不要只相信看到的一些性能对比文章，尤其是不要做无意义的 Hello World级别的测试。
